# 001. Использовать CentralDB для управления локациями

## Статус
Принято

## Контекст
Нужно хранить локации пользователей между сессиями. Варианты: JSON-файлы, SQLite, внешняя БД.

## Решение
Использовать SQLite через класс CentralDB.

## Последствия
- ✅ Работает на Windows без сервера,  
- ✅ Поддерживает несколько локаций на пользователя,  
- ❌ Требует миграций при изменении схемы (пока нет миграций).
